Repository: `solidprivacy-nl/scrub`  
Status: implemented; awaiting GitHub Actions, Hugging Face sync and app verification

## Summary

WP12-FIX2 fixes the blocking Hugging Face runtime failure introduced by WP12-FIX.

The observed runtime error was:

```text
File "/home/user/app/presidio_streamlit.py", line 380
    st.markdown("**Scrub Key laden**")
    ^
IndentationError: unexpected indent
```

Cause:

- `scrub_key_import_ui_block` and `reinsert_ui_block` were generated with two indentation levels.
- They were then inserted directly under:

```text
if solidprivacy_work_mode == "Originele waarden terugzetten":
```

- That made the generated `presidio_streamlit.py` syntactically invalid.

Fix:

- The reinsert-branch blocks now start with exactly one branch indentation level.
- The generated code around `Scrub Key laden` and `Originele waarden terugzetten` should compile correctly.
- The two-mode behavior remains in place.
