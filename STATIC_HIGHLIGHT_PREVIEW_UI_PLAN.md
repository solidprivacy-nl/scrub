# WP42C — Static highlight preview UI planning

Status: completed planning/documentation-only  
Repository: `solidprivacy-nl/scrub`

## 1. Purpose

WP42C plans how the existing `highlight_preview.py` helper may be exposed in a future Streamlit UI without changing UI code now.

The goal is to support online/web validation of document-centric review while preserving the current table-first workflow as the authoritative review surface.

## 2. Decision

The future UI may add a small experimental panel:

```text
Static highlight preview — experimental / read-only
```

This panel may show escaped highlighted text segments generated by `build_static_highlight_preview(...)`.

It must be clearly labelled as:

```text
read-only
non-authoritative
review-assistive only
```

The existing replacement table remains the source of truth for include/exclude/replacement decisions.

## 3. Placement in the existing app

Recommended future placement:

```text
An expander below the existing review guidance and before or near the replacement table.
```

Rationale:

- The preview helps users understand context before editing table rows.
- The table remains nearby for authoritative changes.
- Keeping it in an expander avoids visual overload for users who prefer table-only review.

Do not place it in:

- export/download flow;
- Scrub Key export/import flow;
- reinsert flow;
- DOCX restored output flow;
- PDF-to-TXT reinsert flow.

## 4. UI copy

Suggested Dutch label:

```text
Documentvoorbeeld met markeringen — experimenteel
```

Suggested warning/caption:

```text
Dit voorbeeld helpt om gevonden waarden in context te bekijken. Het is alleen-lezen en niet bepalend voor export. Pas beslissingen aan in de vervangtabel hieronder.
```

Suggested invalid-span warning:

```text
Niet alle markeringen konden veilig worden getoond. Controleer de vervangtabel en technische details.
```

Suggested no-preview message:

```text
Geen veilig documentvoorbeeld beschikbaar voor deze invoer. Gebruik de vervangtabel voor controle.
```

## 5. Allowed UI behavior

Allowed in a later implementation package:

- render escaped text/highlight segments from the helper;
- show Dutch category labels;
- show an accessibility label per highlight;
- show a finding list or summary generated from helper output;
- show invalid span warnings;
- show the helper warning that the table remains authoritative;
- provide a link/reference to row ids if available;
- hide the panel when no safe preview is available.

## 6. Not allowed in this UI step

The first UI integration must not:

- change include/exclude decisions;
- edit replacements;
- mutate review table rows;
- mutate Scrub Key rows;
- affect export eligibility;
- block exports;
- create or delete mappings;
- mark text as sensitive from the preview;
- use external JavaScript or cloud assets;
- render user text as raw HTML;
- process real-data fixtures.

## 7. Data contract

The UI should consume only the helper output:

```text
preview_type
report_only
read_only
non_authoritative
mutation_allowed
export_blocking
scrub_key_changes
segments
valid_spans
invalid_spans
safe_to_render
category_labels
warning
```

Required gate before rendering:

```text
safe_to_render is True
read_only is True
non_authoritative is True
mutation_allowed is False
export_blocking is False
scrub_key_changes is False
```

If any gate fails, the UI must not render highlight markup and should show a warning instead.

## 8. Rendering rule

The future UI must render only `escaped_text` from helper segments.

It must not render raw `text` values as HTML.

Markup may be generated only by trusted app code around escaped text.

## 9. Accessibility rule

The future UI must not rely on color alone.

Minimum display requirements:

- visible category label or legend;
- selected/highlight label text;
- table fallback remains available;
- invalid-span warning is textual;
- high-risk unresolved category is named explicitly.

## 10. Performance boundary

The first UI integration should be limited to small previews.

Recommended boundaries:

```text
small plain text or extracted main text
bounded number of highlight spans
fallback to table-only when preview is too long or invalid
```

No virtualized editor or long-document highlight renderer is approved in this step.
